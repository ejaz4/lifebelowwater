Splash screen

Technical Description

The splashScreen.html file is designed as a splash screen that plays a background video while displaying the organization's name, mission statement, and a list of contributors. The page automatically redirects to home.html after 4 seconds.

Key features

Structured Content
  • Header: Displays the title “SeaGate” with a tagline
  • Logo: A visually prominent SVG image (/images/logo-filled-negative.svg)
  • Organization Mission Statement: Provides details about the UK-based charity’s goal
  • Group Member List: Acknowledges contributors
  • Animated Loading Indicator
    • Uses three animated circles (div.circle) to indicate page loading

Auto-Redirect

  • meta http-equiv="refresh" content="4; url=home.html" Redirects to home.html after 4 seconds

div.circle

Each div class="circle" represents a small bouncing circle in the loader animation

How it works

display: flex; in .loader
  • Aligns the three circles horizontally (justify-content: center;)
  • Adds spacing between them (gap: 20px;)
  • Aligns them at the bottom of the container (align-items: flex-end;)
.circle Styles
  • Shape: border-radius: 50%; to make them circular
  • Animation: animation: moveUpAndDown 2s ease infinite; (applies the bouncing effect)
Bouncing Animation (@keyframes moveUpAndDown)
  • 0% and 100%: The circles remain at their original position
  • 50%: The circles move 50px upwards (translateY(-50px);)
  • The movement creates a bouncing effect
Animation Delays
  • The 2nd circle starts 0.5 seconds later
  • The 3rd circle starts 1 second later
  • This staggered timing creates a wave-like bouncing effect instead of all circles jumping together

Accessibility

Semantic HTML

  • Uses h1, h2, h3, p, and img appropriately

Color Contrast & Readability

  • The dark overlay (rgba(0, 0, 0, 0.5)) ensures text remains readable against the video

Keyboard & Screen Reader Navigation

  • Content is structured logically with clear heading levels

Volunteer

Technical Description

Structured Content

  • Video
  • Charity projects
  • feedback and reviews

Video

  • Uses a video element with autoplay, loop, muted, and playsinline attributes.
  • Video is positioned using CSS (video-volunteer class).

Display: flex is used to position the video so it stays at the top and moves with the size of the tab. justify-content: center and align-items: center are used to center text into the middle of the video.

Volunteering Projects Section

  • Contains multiple elements within .table-container, listing different volunteer projects
  • Each table consists of:
    • Project title (th)
    • Project description (td)
    • Project images (img elements inside .image-container)
    • Project duration and requirements (ul li)
    • "Apply" button (button)

    • Expandable Details Section

    • Uses details and summary for two additional projects

    Feedback and Rating Section

    • Displays volunteer ratings (h2 with rating score)
    • Includes user recent reviews and total amount of reviews (.review containers)
    • Feedback form with a textarea for user suggestions

    Background & Typography

    • Uses Inter Tight font for the main content
    • Background color is rgba(0, 136, 255, 1), a blue shade

    Tables and Layout

    • .table-container uses display: flex; for a side-by-side table layout (desktop)
    • @media (max-width: 647px) changes .table-container to a column layout for mobile

    Buttons

    • .button has a groove border, rounded corners, and changes color on :hover

    Accessibility

    Semantic HTML

    • header, main, and section tags used for logical structure
    • label is used for select elements
    • details and summary enhance usability

    Text and Contrast

    • Most text elements (h1, h2, p, label, summary) use high-contrast colors (rgba(249, 246, 238, 1) on blue background)

    Forms and Input Accessibility

    • textarea includes:
      • maxlength="200"

    Challenges and Lessons Learned

    My main challenge while developing these pages was implementing flexbox, this was mostly due to it being my first time utilizing it into my code. However in the end I believe the struggle was worth it in the end due to me enjoying the learning experience and it allowing for a user friendly design. Another challenge was with the animated loader on the splash screen, this was because my original plan for this feature was to use a gif but in this use case. To overcome this I used 3 divs called circle which I animated using some css. My final challenge was finding a color scheme that fit the theme of the topic (life below water). I decided on a dark blue for the background of my pages, a light blue for tables and similar items, bone white for text and a dark blue and bone white text for buttons then a light gray and black text for when hovering over.
